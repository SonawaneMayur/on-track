# onTrack

A personal, **on-device** family task & routine app for iOS and Android, built
from one codebase. It replaces the printed chore charts: editable tasks per
person, tap-or-voice to complete, calendar awareness, and reminders that fire
with the app closed.

No accounts. No backend. No cloud sync. Data lives on the device. See
[BUILD.md](./BUILD.md) for the full design rationale and non-goals.

## Stack

- **Svelte 5 + Vite + TypeScript** — a light web layer (no React-scale tooling).
- **Capacitor 8** — wraps the web layer into native iOS + Android shells and
  provides the native plugins a pure web app can't.
- **On-device SQLite** (`@capacitor-community/sqlite`) for tasks, completions,
  and the cached calendar. A localStorage backend stands in for browser dev so
  the whole app runs in a normal browser with no device.

Everything is computed from two tables — `tasks` and `completions`. History,
streaks, and "missed" are derived on read; nothing is precomputed or duplicated.

## Features (mapped to BUILD.md phases)

| Phase | Feature | Where |
|------|---------|-------|
| 1 | Editable tasks, tap-to-complete, local notifications | `Today.svelte`, `TaskEditor.svelte`, `services/notifications.ts` |
| 2 | Reports: done/missed, completion rate, streaks, best/worst, per-person & per-section | `Reports.svelte`, `services/reports.ts` |
| 3 | Calendar awareness via private iCal (ICS) feed; silence reminders during events | `CalendarView.svelte`, `services/calendar.ts`, `services/ics-parser.ts` |
| 4 | Command-based voice (STT add/complete/query) + TTS read-aloud | `VoiceButton.svelte`, `services/speech.ts`, `services/voice-intents.ts` |

Design choices that follow the spec directly:

- **"Missed" is lazy-on-read** (no nightly job): an occurrence with no completion
  becomes `missed` once its day has ended / its time + grace has passed.
- **Reminders are concrete, not OS-repeat rules.** We schedule the soonest ~60
  occurrences across a rolling horizon (default 14 days), re-scheduled on every
  app resume. This keeps us under iOS's 64-pending limit *and* lets us skip
  individual reminders that fall inside a calendar event.
- **The secret ICS URL is a credential.** On device it is stored in the iOS
  Keychain / Android Keystore (`@aparajita/capacitor-secure-storage`), never in
  plain preferences. ICS fetch is read-only, outbound HTTPS only.
- **Backup** is a manual JSON export/import (Settings → Backup). No server holds
  your data or any credentials.

## Project layout

```
src/
  lib/
    db/            storage abstraction: SQLite (native) + localStorage (web)
    services/      notifications, calendar/ICS, speech, voice intents, reports, secure storage
    occurrences.ts task → dated occurrence expansion + status resolution (shared by Today & Reports)
    store.ts       central app state + actions
    time.ts        local-date / recurrence helpers
    seed.ts        first-run starter chart
  components/      Svelte UI
capacitor.config.ts
ios/  android/     native shells (generated by Capacitor)
```

## Develop

```bash
npm install
npm run dev        # browser preview (uses the localStorage backend + Web Speech API)
npm run check      # type-check
npm run build      # bundle to dist/
```

## Run on a device

```bash
npm run build
npx cap sync                 # copy web build + plugins into the native projects
npm run open:ios             # opens Xcode  → run on a simulator/device
npm run open:android         # opens Android Studio → run on an emulator/device
```

Requirements: **Xcode** (iOS) and **Android Studio + JDK 17** (Android). iOS uses
Swift Package Manager — no CocoaPods needed.

### Known platform note — iOS voice input

The community speech-recognition (STT) plugin does not yet ship Swift Package
Manager support, so on the SPM-based iOS build it is not linked and voice
*input* is hidden there (the mic button only appears where STT is available).
Everything else on iOS works, **including TTS read-aloud**. Android links all
plugins and has full voice. To enable iOS STT, build iOS via CocoaPods or swap
in an SPM-compatible STT plugin when one is available.

## Privacy & security

- Data never leaves the device except through an explicit user export.
- The only secret is the ICS feed URL, kept in OS secure storage and never logged.
- The app runs no server and accepts no inbound connections.
